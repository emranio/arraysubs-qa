# Stage 14 — Task 06: Related Orders and Refunds

| Key | Value |
|---|---|
| Stage | Admin Subscription Management |
| Module | Admin SPA — Order History card |
| Plugin Coverage | Free |
| Estimated Time | 20 min |
| Depends On | 14-04, Stage 06 (initial + at least one renewal order), one refund processed |

## Objective
Open the **Order History** card on a subscription detail page and verify it lists the parent (Initial) order, every renewal order, and any refund sub-rows. Confirm the columns Order, Date, Status, Total, Refunded, Type, Actions render correctly. Confirm the Total Refunded badge in the card header matches the sum of refund sub-rows. Confirm that the parent → renewal → refund relationship is consistent with what is shown in WooCommerce → Orders.

## Pre-conditions
- Logged in as Administrator.
- A subscription exists that has:
  - 1 Initial order (status Completed) from stage 06 checkout.
  - At least 1 Renewal order (status Completed) generated by the renewal job.
  - At least 1 partial refund processed against either the initial or a renewal order through WooCommerce → Orders.
- If no refund has been processed yet, do so now via WooCommerce → Orders → open the order → Refund button → enter a partial refund amount with a reason.
